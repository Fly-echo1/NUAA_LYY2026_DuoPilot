# DuoPilot v2.0 → v3.0 技术变更文档

> 项目: NUAA 天目启航 · 双车协同循迹系统  
> MCU: TI MSPM0G3507 (Cortex-M0+)  
> 视觉: Canaan K230 (AprilTag 识别)  
> 通信: HC-05 蓝牙 (UART)  
> 惯性: WIT 九轴姿态传感器 (UART)  
> 日期: 2026-05-19

---

## 一、概述

v3.0 在 v2.0 双车超车能力的基础上，引入 **测试模式** 与 **陀螺仪前馈**，增强调试效率与弯道性能。

- **v2.0**: 双车协同，后车 AprilTag 测距跟随 + 开环超车时序
- **v3.0**: 测试模式编译开关（断外设依赖调试）、WIT 陀螺仪 Yaw/Gz OLED 显示、角速度前馈弯道预判算法框架

---

## 二、架构变更

### 2.1 测试模式

新增 `TEST_MODE` 宏，编译时切断对外设依赖，使开发者可在无蓝牙 / 无 IMU 环境下调试核心循迹逻辑。

```c
#define TEST_MODE    // main.c:41

#ifdef TEST_MODE
    OLED_ShowString(0, 0, (uint8_t *)"TEST MODE", 16);
#endif

#ifndef TEST_MODE
    /* 蓝牙 "OV" 超车指令检测 — 编译时排除 */
    ...
#endif
```

| 模式 | 蓝牙检测 | WIT 陀螺仪 | 适用场景 |
|------|---------|-----------|---------|
| 正常模式（无 `TEST_MODE`） | 启用 | 启用 | 双车实车运行 |
| 测试模式（定义 `TEST_MODE`） | **禁用** | 启用 | 单车调试 / 实验室验证 |

### 2.2 陀螺仪角速度前馈

在转向环 PID 输出上叠加角速度前馈项，用于弯道预判：

```

转向环 PID 输出
        │
        └── + ──→ 差速输出
              ↑
        Gz 一阶 IIR 低通
              │
        └── 死区 (|Gz| < 1°/s)
              │
        └── 偏差幅度闸门 (|偏差| ≥ 30)
              │
        └── FF_GAIN (待整定，当前为 0.0)
```

参数当前设 `FF_GAIN = 0.0`，保留算法框架待实车整定。

## 三、新增 / 修改模块

### 3.1 测试模式 (`main.c`)

- `__enable_irq()` 显式调用 — 修复 v2.0 中 SysTick 不触发导致 `mspm0_delay_ms()` 死等的 bug
- 蓝牙 OV 指令检测段以 `#ifndef TEST_MODE` 包裹
- OLED 首行显示 "TEST MODE" 标识

### 3.2 陀螺仪前馈 (`main.c` section 7a)

```
static float gz_filtered = 0.0f;
const float ALPHA    = 0.3f;    // 一阶 IIR 低通系数
const float DEADBAND = 1.0f;    // °/s 死区
const float FF_GAIN  = 0.0f;    // 前馈增益（待整定）
const float FF_DEV_MIN = 30.0f; // 偏差幅度闸门
```

算法分三个阶段:
1. **低通滤波**: `gz_filtered = gz_filtered * 0.7 + wit_data.gz * 0.3`
2. **死区**: 小于 1°/s 的角速度视为静止噪声，置零
3. **偏差闸门**: |灰度偏差| < 30 时（直道）关闭前馈，防止正反馈蛇行

### 3.3 OLED 显示扩展 (`main.c`)

| 行 | v2.0 | v3.0 |
|----|------|------|
| 行 0 | `Dev:+123` | 同上 + 超车阶段 `OV02` |
| 行 1 | (空) | `Y:+123.4` — WIT Yaw 偏航角 |
| 行 2 | `L: 40 R: 40` | 同上 |
| 行 3 | (空) | `Gz:+123` — WIT Z 轴角速度 |
| 行 4-5 | P/I/D 分量 | 同上 |
| 行 6 | SPD | 同上 + K230 距离 |

### 3.4 代码质量修复

**Control.c**
| 项 | v2.0 | v3.0 |
|----|------|------|
| 头文件 | 缺少 `stdint.h`/`stdbool.h` | 显式包含 |

## 五、关键 API 说明

### 主循环时序（~50Hz）

```
 ┌─ 1. 读取灰度传感器
 ├─ 2. 读取 K230 AprilTag 数据
 ├─ 3. [TEST_MODE 关闭时] 轮询蓝牙 "OV" 指令
 ├─ 4. 状态机派发
 │    ├─ OVERTAKING → overtaking_update(), 跳转到末尾
 │    └─ FOLLOWING → 继续执行
 ├─ 5. 灰度偏差计算
 ├─ 6. 距离 PID 计算
 ├─ 7. 转向环 PID
 ├─ 7a. 陀螺仪角速度前馈 (IIR 低通 + 死区 + 偏差闸门)  ← 新增
 ├─ 8. 目标速度合成 (转向差速 + 距离修正)
 ├─ 9. 速度环 PID (编码器反馈)
 ├─ 10. OLED 显示 (含 Yaw/Gz/距离)
 ├─ 11. 蓝牙调试输出
 └─ 12. delay(20ms)
```

### 初始时序修复

v2.0 缺少 `__enable_irq()`，导致 `SysTick_Handler` 不触发 → `mspm0_delay_ms()` 永久等待：

```
v2.0: SYSCFG_DL_init() → SysTick_Init() → OLED_Init() → ... → mspm0_delay_ms() 死等！
v3.0: SYSCFG_DL_init() → SysTick_Init() → __enable_irq() → OLED_Init() → ... → delay_ms 正常工作
```

## 六、回归注意事项

1. **测试模式下蓝牙不可用**: `TEST_MODE` 启用时，HC-05 超车指令段被编译排除，但不影响 UART 调试串口输出
2. **陀螺仪前馈初始关闭**: `FF_GAIN = 0.0`，需实车整定后方生效
3. **WIT 传感器必须连接**: Yaw/Gz 数据来源于 WIT 姿态传感器，断连时 OLED 显示无效值
4. **SysConfig 无需变更**: 所有外设配置与 v2.0 一致
