Improvements · 改进建议

发现的问题与改进方向

CRITICAL · 严重
uart0_send_string 空指针检查错误
str!=0 && str!=0 重复检查 str 本身，应改为 *str!='\0' && str!=NULL。 当前代码偶然工作只因 str 递增后触 NULL 退出循环。
WARNING · 警告
控制频率未达预期
注释声称 ~50Hz，但灰度扫描阻塞 ~8ms + 20ms delay = 实际 ~33Hz。
灰度扫描阻塞过长
delay_cycles(320000) × 8 = ~8ms 纯阻塞等待。
ADVICE · 建议
• 死代码清理 — Grayscale.c 尾部残留大段注释旧代码
• OLED 位置魔数 — 5*6, 7*6 硬编码，改字体需全表重算
• 缓冲区复用 — oled_buffer[64] 同时用于 OLED 和 UART，脆皮设计
• 未生效的驱动程序 — MPU6050, IMU660RB, LSM6DSV16X 编译但不使用，空耗空间
• 缺少 .gitignore — Debug 产物、clangd 缓存被纳入版本管理
• 无看门狗 — MCU 死机无自动恢复机制

Deploy · 部署指南

如何构建 & 运行

01
安装 Code Composer Studio
TI CCS v12+，通过 Resource Explorer 安装 MSPM0 SDK 2.10.00.04
02
导入项目
Project → Import CCS Projects → Browse 选择 car.102 目录 → Finish
03
连接调试器 & 烧录
J-Link 连接 SWCLK(PA20) / SWDIO(PA19) / GND / 3.3V
构建 Ctrl+B · 调试 F11
04
调试：串口查看数据
蓝牙模块接 PA10 (TX)，串口助手 115200 bps 查看：
dev:+123 L: 45 R: 35 cnt:2 SL: 40 SR: 38
接线提醒
Checklist
✅ 左电机: PA17/PA18 + PWM PB8
✅ 右电机: PB9/PA15 + PWM PA14
✅ 编码器 A: PA22/PA21
✅ 编码器 B: PA24/PA23
✅ 灰度: 8路 IO + AD0~AD2 MUX
✅ OLED: SCL PA12 / SDA PA13
✅ 蓝牙: TX→PA10
✅ 蜂鸣器: PA27
✅ J-Link: SWCLK/SWDIO/GND
上电后蜂鸣器响一声、OLED 显示 "Car.101 PID" 即代表系统正常启动。
